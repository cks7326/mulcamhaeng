데이터
  프로그램 사용해야 하는 실제 값
  값 - 단순하지 않다 : 문제
  값 - 복잡하다.

  값을 정리하는 학문 - 자료구조
  정리 후 새롭게 저장하는 곳 -> 어디?
   내 컴퓨터에 문서로 저장하는 시대
   컴퓨터가 문제가 있으면 - 데이터는 삭제 (이번 언론에 보면 타블로? 아이폰 데이터 삭제)
    데이터를 한곳에 안전하게 관리, 사용 - Database 시스템
      : 로컬에 저장하고 다른곳에서 별도로 저장하는 방법(백업)
      응용프로그램은 로컬에 저장한 곳에서 데이터를 사용하고
      문제가 있으면 백업을 확인해서 복구/복원 합니다.
    
    네트워크로 연결이 되고, 데이터 더 복잡해 지면서 - 관계형 데이터 베이스 시스템
     - 회사의 업무용 프로그램 중에 대부분은 관계형 데이터 베이스 시스템
    
    데이터가 너무 많이 쌓여서, 관계형 데이터베이스로도 힘들 경우 생김
     - 조금 더 효율적으로 관리, 사용, 저장 방법
      : 분산 데이터 베이스 시스템
    
    기업용으로 많이 사용

    데이터를 효율적으로 빠르게 전달을 하기 위해서 필요한 방법과 방식
     - 데이터를 빠르게 전달하는 방법 - 필요한 부분만 꼭 집어서 전달
      : 많은 데이터에서 필요한 부분만 문서형식으로 전달 방법 - XML 문서 형식으로 전달
        XML 문서 형식으로 하면 데이터베이스 화 가능하기 때문에
        문서 또는 데이터를 표현하는 방법이 더 발전하게 됨
     -> JSON : JavaScript Object Notation
        클라이언트와 서버가 보통 웹으로 통신을 할 때 서버가 필요한 데이터를 꼭 집어서
        전달하는 표현 방식.
        JavaScript 문법에 영향을 받아서 개발된 가벼운 데이터 표현 방식
        JSON 방식은 : Key - Value Pair 방식
        문서 형식으로 정의 하여 표현하고 파일을 만들 때는 .json 파일 확장자를 이용함
    
    { } [] 키 값
    객체 형식으로 사용하는 것 : 객체 형식 표현방법
    {
        "name" : "홍길동",
        "age" : 20
    }

    배열구조 형식으로 사용한 것 : 배열구조 표현방법
    [
        "name", "홍길동",
        "age", 20
    ]

    객체 형식으로 표현하면서, 필요시 배열 구조를 객체 형식 안에 사용함
    {
        "사번(직원아이디)": 12345678,
        "이름" : "홍길동",
        "기타정보" : [
            {"입사일": "2022-01-21", "집주소": "서울시"},
            {"부서정보": "개발", "메일주소": "aaa@aaa.co.kr"}
        ]
    }
